# 为什么慢
因为innodb中由于事务隔离的限制，不能够像myisam那样将数据行存储起来，需要的时候直接查询，而是每一次都需要扫描表中行来计算，所以当数据量大的时候查询的速度会显的十分慢    
# 统计查询速度排行  
count(字段)<count(主键 id)<count(1)≈count(*)，所以我建议你，尽量使用 count(*)   
* 字段查询最慢是因为innodb需要判断字段的值是否为null，如果为null才会+1去计数，否则是不计入统计的   
* count(主键 id)第二慢是因为InnoDB会遍历整张表，把每一行的id值都取出来，返回给sever层。sever层拿到id后，判断id是不可能为空的，然后累加   
* count(1)Innodb会遍历整个表，但不取值。sever层对于返回的每一行，放一个数字“1“进去，判断是不可能为空的，然后累加。但看count(1)和count(主键 id)可以对比出来前者要快一些，因为从引擎返回id会涉及到解析
数据行，以及拷贝字段值的操作   
* count(*)做了专门的优化，不取值。默认肯定不会为null，直接按行累加
综上，count(*) ≈ count(1) 大于其他的效率   
# 查询太慢解决方法
单独将统计数据在一个表中记录，然后查询这个表
